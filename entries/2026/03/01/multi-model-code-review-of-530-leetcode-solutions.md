# Multi-Model Code Review of 530 LeetCode Solutions

**Date:** 2026-03-01
**Time:** 15:15

## Overview

Ran multi-model-code-review (Claude + Gemini) on all 509 LeetCode solutions generated by multiagent-loop. **Zero review failures.** Results: 103 PASS, 361 CONCERN, 46 BLOCK. Total review time: 9 hours 24 minutes.

The reviews surfaced two systemic issues in the multiagent-loop's code generation:
1. **Wrong function names** — 32 solutions have function names from unrelated LeetCode problems
2. **Trailing spaces in return values** — 5 solutions return strings with trailing spaces that would fail LeetCode's judge

Only **4 solutions** had genuinely broken logic. The other 42 BLOCKs were caused by naming issues, trailing spaces, or reviewer timeouts — not correctness bugs.

### The Wrong Function Name Problem

**32 out of 46 BLOCKs** were caused by solutions using function names from completely unrelated LeetCode problems. Examples:

| Problem | Generated Name | Should Be |
|---------|---------------|-----------|
| cousins-in-binary-tree | `tallestBillboard` | `isCousins` |
| complement-of-base-10-integer | `pancakeSort` | `bitwiseComplement` |
| destination-city | `watched_videos_by_friends` | `destCity` |
| find-winner-on-a-tic-tac-toe-game | `validateBinaryTreeNodes` | `tictactoe` |
| strong-password-checker-ii | `merge_nodes_between_zeros` | `strongPasswordCheckerII` |
| check-if-all-as-appears-before-all-bs | `firstDayBeenInAllRooms` | `checkString` |
| maximum-ascending-subarray-sum | `concatenated_binary` | `maxAscendingSum` |
| largest-perimeter-triangle | `min_area_rect` | `largestPerimeter` |

**Root cause:** The `function_name.txt` files in the LeetCode dataset (`~/data/leetcode/leetcode/`) contain function names that don't match their parent problem directory. This is a data quality issue in the original dataset, not a bug in multiagent-loop. The multiagent-loop faithfully used whatever function name was specified.

**Impact:** The logic is correct in all 32 cases — every solution solves the right problem, it's just the function name that's wrong. These would fail LeetCode submission but are algorithmically sound.

### Trailing Spaces Problem

5 solutions return strings with trailing spaces from the LeetCode problem descriptions:

- **best-poker-hand**: Returns `"Flush "` instead of `"Flush"`
- **categorize-box-according-to-criteria**: Returns `"Both "`, `"Bulky "`, etc.
- **find-winner-on-a-tic-tac-toe-game**: Returns `"A "`, `"B "`, etc.

**Root cause:** The LeetCode problem descriptions in the dataset include trailing spaces in the example outputs (an artifact of web scraping). The multiagent-loop's implementer faithfully reproduced these trailing spaces. The tests also encode the trailing spaces, so everything passes internally but would fail LeetCode's exact-match judge.

### Real Logic Bugs (4 problems)

Only 4 solutions out of 530 had genuinely broken logic:

1. **faulty-sensor**: Logic for determining which sensor is faulty is **inverted** — returns sensor 2 when sensor 1 is faulty and vice versa. Gemini caught this one.

2. **best-poker-hand**: Trailing spaces in return values would fail LeetCode's judge. Logic is correct otherwise.

3. **categorize-box-according-to-criteria**: Same trailing space issue as best-poker-hand.

4. **strong-password-checker-ii**: Two bugs — wrong method name (`merge_nodes_between_zeros`) AND space incorrectly included in the special character set. Trailing spaces in test strings mask the issue.

**Bug rate: 4/530 = 0.75%** — less than 1% of solutions have actual logic errors.

### Model Disagreements

The multi-model approach proved valuable. When Claude and Gemini disagree, it signals areas that need human attention:

| Disagreement Pattern | Count | Example |
|---------------------|-------|---------|
| Claude: CONCERN, Gemini: PASS | ~200 | Claude flags recursion depth, naming issues |
| Claude: BLOCK, Gemini: PASS | ~15 | Claude catches wrong function names |
| Claude: PASS, Gemini: CONCERN | ~30 | Gemini flags integration issues |
| Claude: BLOCK, Gemini: BLOCK | rare | Both agree on serious issues |

Claude was consistently more cautious than Gemini — flagging naming issues, brittle imports (`sys.path.insert`), missing edge cases, and recursion depth risks. Gemini tended to focus on whether the core algorithm was correct and was more lenient on naming.

### CONCERN Analysis (361 problems)

The 361 CONCERNs fall into recurring patterns:

1. **`sys.path.insert` for imports** — Nearly universal. The tester imports from `../implementer/` using path manipulation. Functional but brittle.
2. **Missing edge cases** — Reviewers flag untested inputs (empty arrays, single elements, boundary values).
3. **Recursion depth** — Tree/graph problems using recursive DFS get flagged for potential `RecursionError` on large inputs.
4. **No integration/entry point** — Solutions lack `__init__.py` or module wiring beyond direct import.
5. **Wrong function names** — Some CONCERNs (not BLOCKs) also have mismatched names but the reviewer was less strict.

These are all valid observations but none represent correctness bugs — the solutions work for the given constraints.

## Next Steps

### Actionable Fixes

1. **Fix function_name.txt mapping** in the LeetCode dataset — audit which names don't match their problem directories
2. **Strip trailing spaces** from LeetCode problem descriptions before passing to multiagent-loop
3. **Fix the 4 logic bugs** (faulty-sensor is the most serious — inverted logic)
4. **Consider adding function name validation** to multiagent-loop — cross-check the function name against the problem description

### Systemic Improvements to multiagent-loop

The reviews revealed that the minimal effort pipeline could benefit from:
- **A naming sanity check** — verify the function name makes sense for the problem
- **Output sanitization** — strip trailing spaces from string returns
- **Import structure** — use a proper package layout instead of `sys.path.insert`

### Key Takeaway

**The multi-model review found real issues that 530 successful test runs missed.** Tests passing doesn't mean the code is correct — the tests encoded the same bugs as the implementations (trailing spaces, wrong names). This validates the "Claude Is Your User" philosophy: agents catch different things than automated tests.

**Bug rate: 0.75%** across 530 solutions — the multiagent-loop produces highly reliable code, but multi-model review adds a valuable second layer of verification.
